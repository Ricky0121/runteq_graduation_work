## サービス概要
ストーリー生成アプリ『Memolia』
あなたの思い出を物語にして記録するアプリです。
ユーザーは写真や日記を投稿するだけで簡単に物語を作成することが出来ます。
グループ機能を活用することで家族や友人、恋人と思い出を共有することも出来ます。

## このサービスへの思い・作りたい理由
私は現在実家暮らしで3世帯の家族が一緒に暮らしています。
今後家族が離れ離れになった時にコミュニケーションを取る機会が少なくなってしまうのではないかという悩みから
今回のアプリを作成して、離れた家族や友人、恋人を持つ人にアプリを通じて少しでもコミュニケーションを取る機会を
増やすお手伝いが出来ればと思っています。

## ユーザー層について
メインとなるユーザー層は下記を想定しています。
①単身赴任している社会人(30代〜50代)
　→ 仕事の都合で妻や子供などの家族と離れ離れになっている。
　  仕事関係以外でコミュニケーションを取る機会が減ってしまっている。
　  LINEでは業務連絡のような淡白なやりとりになりがち。
　  家族の「日常」が見えず、孤独を感じやすい。

②上京したての大学生(18歳)
　→ 大学進学を機に一人暮らしをしている。
　  大学で友人はいるが、実家の家族とのコミュニケーションが希薄になってしまっている。
　  親には元気でいることを伝えたいが、電話やLINEは面倒。

③遠距離恋愛をしている恋人(10〜20代)
　→ 仕事の都合や大学への進学、親の都合で引越し、恋人と遠距離恋愛になってしまった。
　  会えない日々でもお互いの今を共有したい.
　  恋人とのコミュニケーションツールの一つとしてそれぞれの生活で思い出を共有して少しでも距離を縮めたい。
　  LINEや通話では出来事の共有がメインにならない。
　  2人だけの空間で思い出を積み重ねたい。

④同じ趣味を持つ友人(全年齢)
　→ 旅行や筋トレ、スポーツなど共有の趣味を持つ友人が思い出を共有
　  相乗効果で互いのモチベーションを高め合っていく。
　  仲間内でだけシェアしたいが、LINEでのグループチャットだけだと振り返りが投稿して終わりになってしまう。
　  続けたい活動を“習慣化”したい。

## サービスの利用イメージ
私たちは日常の中でイベントや出来事があるとスマホで写真を撮って思い出として残しているが、
今回開発するアプリを利用すればこのサービスに写真や簡単な日記をつけることで
ある一定の期間やテーマに沿ってストーリーを自動で生成することが出来るようになり、
アプリ利用者や利用者の家族や恋人に付加価値をつけて思い出が共有出来るようになります。

## ユーザーの獲得について
①単身赴任している社会人(30代〜50代)
　→　家族との心の距離が離れていると感じている
　　 父親がアプリに「写真＋一言メッセージ」を投稿
　　 家族はそれに対していいねで反応
　　 一定の期間(1週間・1ヶ月など)思い出が溜まってきたらAIが自動でストーリーを作成

②上京したての大学生(18歳)
　→ 大学生が「今日の一枚」を投稿
　  親にアプリ内共有、プッシュ通知などでお知らせ

③遠距離恋愛をしている恋人(10〜20代)
　→ 二人だけのプライベートなグループを作成
　  お互いが日常の写真を投稿
　  彼氏・彼女はそれに対していいねで反応
　  月ごとに「2人のAIアルバム」を作成

④同じ趣味を持つ友人(全年齢)
　→ メンバーが写真・メッセージを投稿（例：ジム記録、大会までの練習記録、旅先での発見など）
　  AIが週単位・月単位で活動記録をストーリーとして生成。
　  振り返りの機会が増え、モチベーションが高まっていく。

## サービスの差別化ポイント・推しポイント
類似サービス
みてね【写真共有アプリ】
　→ 限定共有が中心。AI生成要素はありません。
　  今回開発するアプリはストーリーをAIで自動生成にすることでより写真共有に付加価値を持たせます。

MydayAI【生活を管理・改善するためのAI日記サービス】
　→ 毎日の行動・出来事を記録し、日記／行動ログとして残せます。
　  今回開発するアプリは思い出を記録するだけではなく、ストーリーを生成して感情や体験重視になります。

Instagram
　→ Instagramのストーリーはハイライトに追加しない限り投稿から24時間で自動的に削除されます。
　  今回開発するアプリは週・月単位で蓄積してストーリーを構築します。
　  また投稿する目的についてもインスタグラムはフォロワーへの発進が目的ということに対して、
　  今回開発するアプリは家族や恋人、友人への発信が目的になりますので、ターゲットユーザーが異なります。

　  アプリの独自性については、AIを活用した日記アプリは効率化にフォーカスされがちですが、
　  感情表現の支援にフォーカスして作成出来ればと考えております。

## 機能候補
MVPリリースまでに実装したい機能
・共通のトップ画面
・アカウントログインページ
・アカウントの思い出投稿一覧
・アカウント毎の画像アップロード+メッセージ機能
・ストーリーAI自動生成機能
　参考URL：https://www.gluegent.com/blog/2024/05/gpt-4o-api.html
　　　　　　https://qiita.com/MaTTA_RUNTEQ50B/items/3cb46ddf1be17899df00

MVPリリース時点では個人で写真+メッセージからストーリーの自動生成が出来るところまでを実装します。

本リリースまでに実装したい機能
・メールやSNSによるログイン認証機能
・プライベートグループ機能生成
・グループ招待機能
・思い出一覧のタイムライン表示
・いいね・コメント機能
・指定ユーザー思い出投稿時のプッシュ通知機能
・生成AIプロンプト調整作業(感情表現を支援する点を重視したものに調整)
・管理者画面の整備
